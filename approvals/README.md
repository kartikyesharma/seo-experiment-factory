# Manual gates

The deployment script requires three human-created markers per site:

- `approvals/<site>/domain.approved` after the domain is purchased and DNS ownership is confirmed.
- `approvals/<site>/pdf.approved` after the final PDF is reviewed for accuracy, independence wording, links, and price.
- `approvals/<site>/launch.approved` after analytics, payment, delivery, policies, and a real test purchase are approved.

Marker files are intentionally not generated by scripts. Put the reviewer, timestamp, scope, and notes inside each file for an audit trail. Approval does not bypass placeholder checks: the deployment still refuses example domains and payment links.
